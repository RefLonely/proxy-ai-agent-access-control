# 电网卫士：面向新型电力系统的分布式智能代理自主访问控制与内生安全防御平台

## 项目概述

**项目名称**：电网卫士：面向新型电力系统的分布式智能代理自主访问控制与内生安全防御平台

本项目针对2026-A-ST网络空间与信息安全赛项要求，聚焦代理式AI(Agentic AI)自主访问控制在**新型电力系统场景**中的前沿技术难题，重点攻克：
1. **多代理协作中的信任边界动态维护**问题
2. **代理决策中的幻觉抑制与安全对齐**问题

### 应用场景

本项目专门为新型电力系统场景设计，覆盖以下关键应用：
- **分布式能源接入安全**：光伏、风电等分布式能源设备的安全接入与控制
- **智能配电系统保护**：配电网自动化设备的访问控制与异常检测
- **电力监控系统防护**：SCADA、DCS等监控系统的安全审计与访问管理
- **用户侧涉网安全**：负荷聚合商、虚拟电厂等用户侧主体的权限管控

### 电网卫士平台定位

作为面向新型电力系统的核心安全平台，本项目提供：
1. 分布式代理通信与访问控制
2. 动态信任边界维护与安全评估
3. LLM决策幻觉抑制与安全基模对比
4. 内生安全防御与异常行为检测
5. 电力协议解析与实时异常检测

## 问题描述

### 1. 电网场景多代理协作中的信任边界动态维护问题

在新型电力系统中，多代理协作场景极为复杂：光伏代理、风电代理、配电自动化代理、用户侧负荷聚合商代理等需要协同完成电力调度任务。传统静态访问控制模型存在以下局限性：

- **静态边界无法适应动态电力系统**：电力生产流程随天气、负荷、设备状态动态变化，静态ACL无法适配频繁的协作关系变更
- **跨域信任难以量化管理**：发电侧、输电侧、配电侧、用户侧之间的代理协作需要动态调整信任级别，缺乏细粒度的信任评估机制
- **大规模代理协作扩展性差**：中心化授权模型在数以万计的分布式终端代理场景中成为性能瓶颈

### 2. 电网卫士平台中的幻觉抑制与安全对齐问题

代理式AI基于大语言模型进行自主访问决策时，面临：

- **恶意协议注入导致权限越界**：攻击者通过构造恶意电力协议数据（如MODBUS、DL/T 645协议），诱导代理做出违规授权决策
- **幻觉导致电力安全规则偏离**：LLM可能对电力业务场景产生错误理解，偏离《电力监控系统安全防护规定》等预设安全要求
- **安全对齐难以动态维护**：随着电力调度策略变化，安全策略需要动态更新，对齐成本高昂

## 创新解决方案

### 方案一：基于动态信念图的电网卫士信任边界维护

我们提出一种创新的**动态信念图(Dynamic Belief Graph, DBG)** 架构，用于新型电力系统中的分布式多代理信任边界维护：

1. **信念图表示**：每个代理维护一个本地信念图，节点表示电力代理实体（光伏代理、PLC代理、SCADA代理等），边表示信任关系，权重表示信任评分(0-1)
2. **信念传播机制**：基于电力调度流程上下文，动态更新信任评分，通过共识算法实现分布式一致性
3. **自适应信任边界**：根据信任评分动态调整访问权限，设置符合电力安全等级保护要求的多级信任阈值，实现弹性边界控制
4. **异常信任衰减**：检测到电力协议异常、访问违规等行为时，自动衰减相关代理的信任评分，快速收缩信任边界

### 方案二：基于电网安全基模对比的幻觉抑制与对齐框架

我们设计了**电网安全基模对比(Power System Security Schema Comparison, PSSC)** 框架，有效抑制LLM幻觉并保持与电力安全规则的对齐：

1. **电网安全基模构建**：将电力访问控制规则（如《电力监控系统安全防护规定》）抽取为结构化的安全基模，包含主体、客体、操作、条件四元组
2. **双路径决策验证**：
   - 路径1：LLM生成候选访问决策
   - 路径2：将候选决策与安全基模进行嵌入空间对比，计算一致性分数
   - 仅当一致性分数高于阈值时，决策才会被执行
3. **动态基模更新**：基于电力调度策略变化和环境反馈，自动学习更新安全基模，保持动态对齐

### 方案三：内生安全防御与异常行为检测

集成内生安全防御机制，对电力代理的通信行为进行实时监控：
- 电力协议解析与异常检测
- 访问模式分析与异常识别
- 分布式协同防御与响应

## 快速开始

### 环境要求
- Python 3.8+
- 依赖包：`networkx`, `numpy`, `scipy`, `dataclasses-json`

### 安装运行

```bash
# 克隆项目
git clone https://github.com/zhoulg/proxy-ai-agent-access-control.git
cd proxy-ai-agent-access-control

# 安装依赖
pip install -r requirements.txt

# 如果遇到外部管理环境错误(Debian/Ubuntu)，可以使用：
# pip install -r requirements.txt --break-system-packages

# 运行示例
python examples/dynamic_trust_example.py
python examples/hallucination_suppression_example.py
```

## 技术先进性

本项目结合了以下最新研究进展：

1. **代理式AI架构**：基于自主代理实现分布式访问控制决策，符合AI技术发展趋势
2. **图信任传播**：基于图结构进行信任关系建模和传播，支持大规模多代理协作
3. **对比学习**：利用嵌入对比实现安全基模匹配，有效抑制幻觉
4. **分布式共识**：基于部分同步共识实现信任状态一致性，避免单点故障

## 实用性

针对新型电力系统实际需求：

- **协议支持**：支持Modbus、OPC UA、DL/T 645、IEC 61850等电力通信协议
- **实时性要求**：单决策延迟<10ms，满足电力调度系统的实时性要求
- **兼容性**：兼容现有电力监控系统（SCADA、DCS、EMS等），无需大规模改造
- **合规性**：提供完整的审计日志，符合等保要求和电力监控系统安全防护规定
- **分布式部署**：支持边缘计算部署，满足海量分布式能源设备接入需求
