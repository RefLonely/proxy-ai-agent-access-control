### 1.2 处理流程

```
1. 访问请求到达
   ↓
2. DBG模块：计算请求代理的信任评分
   ↓
3. 如果信任评分低于最低阈值 → 直接拒绝
   ↓
4. 信任评分通过 → LLM生成候选访问决策
   ↓
5. SSC模块：将候选决策与安全基模比对，计算一致性得分
   ↓
6. 如果得分高于阈值 → 接受决策，执行访问，更新信念图
   ↓
7. 如果得分低于阈值 → 拒绝决策，记录日志，告警管理员
```

## 2. 核心功能模块

### 2.1 动态信念图(Dynamic Belief Graph, DBG)

#### 2.1.1 信任评估机制

**核心思想：**
每个代理分布式维护一个信念图，节点代表电力代理实体，边代表信任评分。通过信念传播聚合直接信任和间接信任，根据信任评分动态调整访问权限，检测到异常时自动衰减信任收缩边界。

**信任聚合公式：**
```
Trust(i,j) = α * DirectTrust(i,j) + (1-α) * Σ(Trust(i,k) * Trust(k,j))
```

**信任更新机制：**
- 成功交互：增加信任评分
- 失败交互：减少信任评分
- 异常行为：快速衰减信任评分

**信任边界伸缩：**
- 信任评分 ≥ 0.8：完全信任 → 允许
- 0.5 < 信任评分 < 0.8：部分信任 → 需要二次验证
- 0.2 < 信任评分 ≤ 0.5：低信任 → 限制敏感操作
- 信任评分 ≤ 0.2：不信任 → 拒绝

#### 2.1.2 信任传播算法

**传播过程：**
1. 初始化每个代理的直接信任
2. 计算间接信任
3. 聚合信任分数
4. 收敛条件：信任分数变化小于阈值

**收敛速度：**
- 通常3-5轮迭代即可收敛
- 每轮迭代时间：< 1ms

### 2.2 电网安全基模对比(Power System Security Schema Comparison, PSSC)

#### 2.2.1 安全基模管理

**安全基模表示：**
```
Schema = (Subject, Object, Action, Condition)
```

**安全基模示例：**
```
{
  "name": "拒绝外部写入PLC",
  "subject_pattern": ".*",
  "object_pattern": "plc|programmable.*logic",
  "action_pattern": "write|set|modify",
  "condition_expr": "domain != 'local' OR trust < 0.8",
  "allow": false
}
```

**基模管理功能：**
- 基模创建和编辑
- 基模验证和测试
- 基模版本控制
- 基模部署和更新

#### 2.2.2 双路径验证机制

**验证流程：**
1. LLM生成候选访问决策
2. 提取决策的主体、客体、操作、条件
3. 将决策与安全基模进行比对
4. 计算一致性得分
5. 根据得分决定是否接受决策

**一致性评估：**
```
Score = w1 * StructureMatch(decision, schema) + w2 * CosineSim(emb(decision), emb(schema))
```

**验证结果：**
- 得分 ≥ 0.8：接受决策
- 0.5 < 得分 < 0.8：需要二次验证
- 得分 ≤ 0.5：拒绝决策
