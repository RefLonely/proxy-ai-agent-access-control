| 项 | 内容 |
|----|------|

## 2. 问题背景与研究意义

### 2.1 行业背景

随着新型电力系统建设的推进，分布式能源接入、配电网自动化、虚拟电厂等技术的快速发展，电力系统正从传统的集中式架构向分布式多代理架构演进。这带来了新的安全挑战，特别是代理式AI自主访问控制问题。不同类型的电力代理（如光伏代理、PLC代理、SCADA代理、负荷聚合商代理）需要动态协作完成电力调度任务，传统静态访问控制机制已无法适应这种动态协作需求。

### 2.2 科学问题

本项目聚焦两个当前代理式AI领域的前沿开放难题：

#### 问题一：多代理协作中的信任边界动态维护

**问题描述：**
- 传统静态ACL/ABAC模型无法适应工业环境中频繁变化的协作关系
- 跨域协作中，如何量化和动态调整对新代理的信任程度缺乏有效方案
- 中心化授权在大规模多代理场景下存在性能瓶颈

**现有技术支持：**
- 多代理信任理论已有基础研究
- 基于图的信任传播方法已有探索
- 零信任架构提出了"永不信任，始终验证"理念

> ✅ 理论基础完备，工程方案有待针对动态场景优化

#### 问题二：代理决策中的幻觉抑制与安全对齐

**问题描述：**
- 基于LLM的代理容易产生幻觉，可能做出违反安全策略的决策
- 提示注入攻击可以诱导代理越权授权，现有防御方法不完全适配自主决策场景
- 基于微调的对齐方法更新成本高，无法适应工业环境策略快速变化

**现有技术支持：**
- LLM幻觉问题已有广泛研究，多种抑制方法被提出
- 检索增强(RAG)和自一致性检查可减少幻觉
- 双路径验证思路在近期研究中被提出

> ✅ 方向已有探索，针对访问控制场景的专用方案有待研究

## 3. 创新方案设计

### 3.1 方案一：动态信念图(Dynamic Belief Graph, DBG)

**核心思想：**
每个代理分布式维护一个信念图，节点代表电力代理实体（光伏代理、PLC代理、SCADA代理等），边代表信任评分。通过信念传播聚合直接信任和间接信任，根据信任评分动态调整访问权限，检测到异常时自动衰减信任收缩边界。

**关键算法：**

1. **信任聚合公式**
   ```
   Trust(i,j) = α * DirectTrust(i,j) + (1-α) * Σ(Trust(i,k) * Trust(k,j))
   ```
   *技术支持：* 图信任传播方法已有研究基础 ，加权聚合思想成熟。

2. **异常衰减机制**
   ```
   After anomaly: Trust(i,j) = Trust(i,j) * γ^n
   ```
   *技术支持：* 指数衰减是常用的信任惩罚方法，在声誉系统中广泛应用 。

3. **多级信任边界**
   - `Trust > 0.8`: 完全信任 → 允许
   - `0.5 < Trust < 0.8`: 部分信任 → 需要二次验证
   - `0.2 < Trust < 0.5`: 低信任 → 限制敏感操作
   - `Trust < 0.2`: 不信任 → 拒绝

**创新性：**
- 分布式架构，无需中心化授权服务器
- 自适应动态调整，异常自动收缩边界
- 收敛快速，满足工业实时性要求

**现有工作对比：**

| 方案 | 分布式 | 动态自适应 | 异常自动响应 |
|------|---------|-------------|---------------|
| 静态ACL | ❌ | ❌ | ❌ |
| ABAC | ❌ | ⚪ 部分支持 | ❌ |
| 中心化声誉 | ❌ | ✅ | ⚪ 延迟 |
| 本文DBG | ✅ | ✅ | ✅ |

### 3.2 方案二：电网安全基模对比(Power System Security Schema Comparison, PSSC)

**核心思想：**
采用双路径验证架构：LLM生成候选决策后，独立将决策与预定义的电力系统安全基模进行结构匹配+嵌入空间比对，仅当一致性得分高于阈值时才执行决策。

**关键算法：**

1. **安全基模表示**
   ```
   Schema = (Subject, Object, Action, Condition)
   ```
   *技术支持：* 基于四元组的访问控制表示与ABAC模型一致，理论成熟 。

2. **一致性评分计算**
   ```
   Score = w1 * StructureMatch(decision, schema) + w2 * CosineSim(emb(decision), emb(schema))
   ```
   *技术支持：*
   - 结构匹配是访问控制检查常规方法
   - 嵌入空间余弦比对在语义匹配中广泛应用，Sentence-BERT已验证有效性

3. **决策规则**
   - `Score > threshold`: 接受决策
   - `Score < threshold`: 拒绝并重生成

**创新性：**
- 不依赖LLM微调，安全策略更新灵活
- 双路径验证，幻觉阻断准确率高
- 增量更新容易，适应工业环境策略变化

**现有工作对比：**

| 方案 | 无需微调 | 动态更新 | 幻觉阻断 |
|------|----------|----------|----------|
| RLHF微调 | ❌ | 慢 | ✅ |
| RAG检索 | ⚪ | 中 | ⚪ |
| 自一致性 | ✅ | 快 | ⚪ |
| 本文SSC | ✅ | 快 | ✅ |


### 4.2 处理流程

```
1. 访问请求到达
   ↓
2. DBG模块：计算请求代理的信任评分
   ↓
3. 如果信任评分低于最低阈值 → 直接拒绝
   ↓
4. 信任评分通过 → LLM生成候选访问决策
   ↓
5. SSC模块：将候选决策与安全基模比对，计算一致性得分
   ↓
6. 如果得分高于阈值 → 接受决策，执行访问，更新信念图
   ↓
7. 如果得分低于阈值 → 拒绝决策，记录日志，告警管理员
```
