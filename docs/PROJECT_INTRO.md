# 电网卫士：面向新型电力系统的分布式智能代理自主访问控制与内生安全防御平台

---

## 项目简介

本项目面向**代理式AI自主访问控制**前沿方向，针对新型电力系统多代理协作安全场景，聚焦解决两个核心技术难题：**(1) 多代理协作中的信任边界动态维护问题；(2) 代理决策中的幻觉抑制与安全对齐问题**。

我们提出两个创新方案：
- **动态信念图(Dynamic Belief Graph, DBG)**：分布式维护信任关系，支持自适应边界伸缩
- **电网安全基模对比(Power System Security Schema Comparison, PSSC)**：双路径验证架构，有效抑制LLM幻觉，保持安全对齐

项目完整实现了可运行原型系统，在新型电力系统访问控制场景下，相比传统方案具有更好的动态适应性和安全性。

---

## 项目背景

### 行业背景

随着新型电力系统建设的推进，分布式能源接入、配电网自动化、虚拟电厂等技术的快速发展，电力系统正从传统的集中式架构向分布式多代理架构演进。这带来了新的安全挑战，特别是代理式AI自主访问控制问题。不同类型的电力代理（如光伏代理、PLC代理、SCADA代理、负荷聚合商代理）需要动态协作完成电力调度任务，传统静态访问控制机制已无法适应这种动态协作需求。

1. **动态性**：电力生产协作关系随天气、负荷、设备状态动态变化，传统静态ACL无法适应
2. **开放性**：跨域协作日益频繁，需要灵活的信任评估机制
3. **自主性**：代理式AI赋予系统自主决策能力，但也带来幻觉和安全对齐问题

### 技术难题

当前代理式AI领域面临两个突出前沿难题：

**难题一：多代理协作中的信任边界动态维护**
- 传统静态信任边界无法适应电力系统的动态协作需求
- 中心化授权模型在大规模多代理场景下扩展性差
- 异常发生后需要人工干预才能收缩信任边界，响应滞后

**难题二：代理决策中的幻觉抑制与安全对齐**
- LLM天生存在幻觉问题，可能做出违反电力安全策略的决策
- 提示注入攻击可以诱导代理越权授权
- 基于微调的对齐方法更新成本高，无法适应电力系统策略快速变化

## 核心功能

### 1. 分布式信任边界动态维护

| 功能 | 描述 |
|------|------|
| 信念图分布式存储 | 每个代理本地维护信念图，无需中心化授权服务器 |
| 动态信任评分计算 | 结合直接交互经验和间接信念传播计算信任评分 |
| 自适应边界伸缩 | 根据信任评分自动调整访问权限，异常发生时快速收缩 |
| 分布式共识一致性 | 基于轻量级共识算法保证信任状态最终一致性 |

### 2. 电网安全基模对比(PSSC)

| 功能 | 描述 |
|------|------|
| 结构化安全基模 | 将电力访问控制规则表示为(主体-客体-操作-条件)四元组安全基模 |
| 双路径验证决策 | LLM生成候选决策后，独立进行安全基模比对验证 |
| 嵌入空间一致性比对 | 结合语义嵌入和结构匹配，准确识别幻觉决策 |
| 动态基模更新 | 支持增量式安全策略更新，无需重新训练模型 |

### 3. 新型电力系统访问控制

| 功能 | 描述 |
|------|------|
| 分级权限响应 | 支持允许→挑战→限制→拒绝→隔离五级响应，兼顾安全与生产 |
| 全链路审计日志 | 记录每一次访问决策过程，满足合规要求 |
| 低延迟决策 | 单请求决策<10ms，满足工业实时性要求 |
| 兼容现有协议 | 支持Modbus、OPC UA、DL/T 645、IEC 61850等电力通信协议接入 |

### 模块交互流程

```
访问请求 → 感知代理收集上下文 → 
           ↓
        信任评估 (DBG) → 计算请求代理信任评分 →
           ↓
        决策生成 → LLM生成候选访问决策 →
           ↓
        安全对齐验证 (PSSC) → 比对安全基模 → 验证通过? 
           ↓              ↓
        执行决策         拒绝决策
           ↓
        更新信念图 → 反馈学习
```

### 关键技术点

**动态信念图(DBG)核心算法：**

1. **信任聚合公式**：
   ```
   Trust(i,j) = α * DirectTrust(i,j) + (1-α) * Σ(Trust(i,k) * Trust(k,j))
   ```
   其中α为直接信任权重，通过信念传播聚合邻居信任信息

2. **异常衰减机制**：
   ```
   当检测到异常行为：Trust(i,j) = Trust(i,j) * γ^n
   其中γ < 1 为衰减因子，n为异常次数
   ```

3. **边界阈值**：
   - 信任评分 > 0.8：完全信任，允许所有授权操作
   - 0.5 ~ 0.8：部分信任，需要二次验证
   - 0.2 ~ 0.5：低信任，限制敏感操作
   - < 0.2：不信任，拒绝访问

**电网安全基模对比(PSSC)核心算法：**

1. **安全基模表示**：
   ```
   Schema = (Subject, Object, Action, Condition)
   ```

2. **一致性评分计算**：
   ```
   Score = w1 * StructureMatch(decision, schema) + w2 * CosineSim(emb(decision), emb(schema))
   ```
   结构匹配+语义嵌入比对双重验证

3. **决策规则**：
   - Score > threshold: 接受决策
   - Score < threshold: 拒绝并重生成
