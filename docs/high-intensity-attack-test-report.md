## 测试4：五级响应机制覆盖测试

五级响应定义（`DecisionOutcome`）：

```python
ALLOW     # 允许 - 低风险，直接通过
CHALLENGE # 挑战 - 中风险，需要二次验证
LIMIT     # 限制 - 中高风险，允许访问但限制权限并记录审计
DENY      # 拒绝 - 高风险，拒绝本次访问
ISOLATE   # 隔离 - 极高风险，拒绝访问并隔离请求源节点
```

覆盖情况：

| 级别 | 代码处理 | 审计日志 | 额外动作 | 测试状态 |
|------|----------|----------|----------|----------|
| ALLOW | ✅ 完成 | ✅ 记录 | - | 通过 |
| CHALLENGE | ✅ 完成 | ✅ 记录 | - | 通过 |
| LIMIT | ✅ 补全完成 | ✅ 记录 | - | 通过 |
| DENY | ✅ 完成 | ✅ 记录 | - | 通过 |
| ISOLATE | ✅ 补全完成 | ✅ 记录 | 自动收缩信任边界 | 通过 |

**ISOLATE额外处理**：检测到极高风险请求时，自动对源代理执行信任边界收缩，所有指向该源的信任评分乘以衰减因子0.5，实现快速隔离。
